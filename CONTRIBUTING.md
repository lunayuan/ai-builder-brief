# Contributing to AI Builder Brief

Thanks for your interest in contributing!

## Ways to Contribute

- **New example configs** — Built a setup for a specific AI niche? Add it to `config/examples/`
- **Prompt improvements** — Better synthesis, sharper implications? Edit files in `prompts/`
- **Bug reports** — Open an issue describing what happened and what you expected
- **Feature ideas** — Open an issue with your use case

## Guidelines

1. **Keep it AI-focused.** This tool is for AI builders. Generic industry support lives in [market-intel](https://github.com/lunayuan/market-intel).
2. **No personal data in PRs.** Config files in `config/examples/` should use placeholder emails and generic descriptions.
3. **Plain English prompts.** The `prompts/` folder uses natural language, not code. Keep it readable.
4. **Test your changes.** Run the skill with your modifications and include a sample output if possible.

## Submitting a PR

1. Fork the repo
2. Create a branch (`git checkout -b my-improvement`)
3. Make your changes
4. Open a PR with a clear description of what changed and why

## Code of Conduct

Be respectful. We're all here to learn and build.
